## AES File Encryptor (tkinter)

### 运行
- 安装依赖：`uv sync` 或 `pip install -r <(uv pip compile pyproject.toml)`（若未用 uv，可直接 `pip install cryptography`）。
- 启动程序：`python main.py`。

### 使用说明
- 密钥框输入 32/48/64 个十六进制字符（对应 AES-128/192/256）；可点击 **生成 256 位密钥** 自动生成随机密钥，再用 **复制密钥** 快速复制。
- **文件加密**
	- 选择明文文件 → 点击 “开始加密”，生成 `<原文件名>.enc`，文件头含随机 IV，内容为 AES-CBC + PKCS7 密文。
- **文件解密**
	- 选择密文文件 → 点击 “开始解密”，若后缀为 `.enc` 则去掉后缀输出，否则追加 `.dec`。
- **文本加解密**（Base64 格式）
	- 输入明文 → 点击 “明文加密 →”，密文框会显示 Base64 编码的 `IV + ciphertext`。
	- 在密文框输入 Base64 → 点击 “← 密文解密”，明文框显示解密结果（UTF-8）。
- **编码模式**
	- “字符表”模式：使用 64 长度字母表一一映射 Base64 符号。
	- “短语表”模式：将 Base64 的 6bit 值映射为短语集合，按权重随机挑选短语；解码时用短语→值恢复。
	- 两种模式可在文本加解密区切换，文件加解密仍使用二进制，不受影响。
- **自定义字符表 JSON**
	- 点击 “加载字符表 JSON” 导入；结构为数组或含 `profiles` 数组，每项需 `name` 与 64 长度的 `alphabet`，可选 `pad`（默认 `=`）。
	- 示例同上。
- **自定义短语表 JSON**
	- 点击 “加载短语表 JSON” 导入；结构为数组或含 `phrase_profiles` 数组。
	- 每个 profile：`name`，`entries`；其中 `entries` 为 0~63 的对象 `{ "value": int, "phrases": [["短语", 权重], ...] }`。
	- 短语需全局唯一，必须覆盖全部 0~63 值；权重为正整数，用于控制短语出现概率。
	- 示例见 `phrase_profiles_example.json`（内置同款已默认加载）。

### 格式约定
- 密文文件格式：前 16 字节为 IV，后续为 AES-CBC 密文（PKCS7 填充）。
- 密钥管理需自行妥善保存；解密必须使用与加密相同的密钥。
